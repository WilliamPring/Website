My Linux Machine: Arch and i3

Ongoing since ~2015

I switched to Linux because I stopped trusting Windows. Not in any dramatic way, just a slow build-up of questions I couldn't answer. What is this process doing? Why is this phoning home? What exactly changed after that overnight update? Windows made it genuinely difficult to know what was happening on your own machine, and the more I thought about it the more that bothered me. I wanted an OS I actually controlled, one I could understand top to bottom and shape to work the way I wanted rather than the way Microsoft wanted.

I started with Ubuntu because that's where most people start. It was an improvement but it still had that same quality of things happening in the background that I didn't fully understand. I kept reading, kept going deeper, and eventually landed on Arch.

Why Arch

Arch doesn't hold your hand. There's no graphical installer, no defaults chosen for you. You boot into a terminal and build up from nothing: partition the disk yourself, install a bootloader, configure the network, pick a desktop environment. Most people hear that and want nothing to do with it. For me it was exactly the point.

When you build the system piece by piece you actually know what you built. Every package is there because I put it there. Every config file has been opened and read. Nothing is running that I didn't choose to install. If something breaks I have a real shot at knowing why, because I was there when it went together. The Arch Wiki is also genuinely some of the best technical documentation I've come across, useful well beyond Arch itself.

The Setup

I run i3 as my window manager. It's a tiling WM so windows don't float around overlapping each other. They tile, slotting into the layout automatically. I almost never touch the mouse. Once you get used to moving through everything by keyboard it's hard to go back.

The status bar is i3status-rust sitting at the top showing what I need to know. I've got three monitors: one primary on HDMI, one to the right on DVI, one to the left on DisplayPort. i3-resurrect handles saving and restoring workspace layout so I don't have to rebuild it after a reboot.

My shell is Zsh with the Starship prompt. Starship only shows information that's actually relevant given what directory you're in: git branch, exit codes, language versions. Antigen handles plugin management. For the terminal I use Kitty, which has GPU rendering and is noticeably snappier than most alternatives. I've had urxvt around for years as a fallback but Kitty is the daily driver now.

Everything I edit goes through Vim. The color scheme across the whole setup is Catppuccin-inspired, configured in .Xresources. Rofi handles app launching. For file management I've been moving from Ranger to Yazi. Both are terminal file managers but Yazi is more actively developed and feels better to use.

Audio runs through PipeWire with EasyEffects for DSP processing. I have a Jack/PipeWire patchbay so I can route audio wherever I need it, with jack_mixer for mixing control. This ends up being important for the passthrough setup. Media playback is MPV with Vulkan rendering and hardware decode. Tmux for terminal multiplexing with a Ctrl-a prefix and Vim-style pane navigation. Newsboat for RSS.

The repo includes an install.sh that handles a fresh Arch install, 112 pacman packages and 8 from the AUR. There's also a cp.sh that syncs the live configs back into the repo before committing so the dotfiles stay current with how things are actually running.
